웹 대화 다운로더

claude.ai · ChatGPT · Gemini의 대화를 로컬 세션 파일로 가져와, 터미널에서 이어서 대화합니다.

01처음 한 번만: 호스트 연결

확장은 브라우저 안에 있어서 파일을 직접 만들 수 없습니다. 파일을 쓰는 역할은 로컬 프로그램(네이티브 호스트)이 맡고, Chrome이 필요할 때 알아서 실행합니다. 터미널에서 아래를 한 번 실행하세요.

cd ~/Desktop/Projects/web-chat-downloader
npm install && npm run build
node dist/cli.js install-host

확장 ID는 자동으로 찾습니다. 서버를 따로 켜둘 필요가 없고, 평소에 떠 있는 프로세스도 없습니다.

02대화 가져오기

claude.ai · ChatGPT · Gemini의 대화 페이지에서 확장 아이콘을 누르면 됩니다.

  • 이 대화 가져오기 — 지금 보고 있는 대화 하나만
  • 전체 동기화 — 그 서비스의 대화를 순서대로 전부
  • 선택 가져오기 — 목록에서 체크한 것만

이미 가져온 대화는 목록에 ✓로 표시됩니다. 목록은 10분간 저장해두고 재사용하며, 대화 N을 클릭하면 즉시 새로 불러옵니다.

03터미널에서 이어가기

가져온 대화 아래에 실제 실행할 명령이 그대로 보입니다. 누르면 복사되니 터미널에 붙여넣으세요.

cd ~/Desktop/Archive/web-chats && claude --resume <세션ID>

웹에서 하던 대화 맥락 그대로 이어집니다.

04아이콘 뱃지

숫자 · ✓ · !

동기화 중에는 진행률(%)이, 문제없이 끝나면 ✓가, 실패한 항목이 있으면 !가 잠깐 표시됩니다. 팝업을 닫아도 동기화는 계속됩니다.

자주 묻는 것

같은 대화를 다시 가져오면 중복되나요?

아니요. 대화별로 세션을 기억해서 같은 파일을 갱신합니다. 웹에서 대화를 더 이어간 뒤 다시 가져오면 최신 내용으로 덮어씁니다. 다만 로컬에서 --resume으로 이어간 내용은 덮일 수 있습니다(웹을 원본으로 봅니다).

"요청이 너무 많다"는 메시지가 떴어요.

서비스가 일시적으로 요청을 제한한 것입니다. 확장은 이 신호를 받으면 즉시 멈추고 그때까지 받은 목록은 살려둡니다. 1~2분 뒤 다시 열어주세요.

목록이 안 뜨고 회색 줄만 보여요.

불러오는 중입니다. 대화가 많은 계정은 몇 초 걸립니다. 한 번 불러오면 10분간 저장해두므로 다음부터는 즉시 뜹니다.

자동 동기화는 언제 도나요?

설정에서 켤 수 있고, 해당 서비스의 탭이 열려 있을 때만 동작합니다. 대화 데이터는 로그인된 페이지 안에서만 읽을 수 있기 때문입니다.

첨부와 이미지는요?

이미지는 세션에 함께 담겨 대화에서 바로 보이고, 그 외 파일은 attachments/ 폴더에 저장되며 대화에는 파일명이 남습니다.
