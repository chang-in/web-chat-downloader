# 기여 안내

관심 가져주셔서 고맙습니다. 작은 수정이라도 환영합니다.

## 개발 환경

```bash
npm install
npm run build      # dist/cli.js — 네이티브 호스트가 이 파일을 실행합니다
npm test           # vitest
npx tsc --noEmit   # 타입 검사
```

확장을 직접 시험하려면:

```bash
node dist/cli.js install-host   # 확장 ID는 자동으로 찾습니다
```

`chrome://extensions` → 개발자 모드 → 압축해제된 확장 프로그램 로드 → `extension/` 선택.

## 구조

| 경로 | 역할 |
|---|---|
| `src/adapters/` | 서비스별 응답을 공통 형식으로 정규화 |
| `src/core/` | 세션 파일 작성(Claude·Codex), 첨부 저장, 중복 방지 |
| `src/native-host.ts` | 브라우저 확장과 stdin/stdout으로 주고받는 호스트 |
| `extension/` | Chrome 확장(팝업·설정·백그라운드·콘텐츠 스크립트) |

핵심 원칙 하나: **세션 형식은 추측하지 않습니다.** 실제 세션 파일을 여러 개 비교해 필드를 정하고, 가능하면 실제로 재개되는지 확인한 뒤 반영합니다.

## 바꿀 때

- 커밋 메시지: `<type>: <설명>` (`feat`·`fix`·`refactor`·`docs`·`test`·`chore`)
- 새 기능은 이슈로 먼저 이야기해주시면 방향이 어긋나는 일을 줄일 수 있습니다
- 확장 코드는 자동 테스트가 닿지 않으므로, 바꿨다면 **실제 브라우저에서 확인**한 내용을 PR에 적어주세요
- 의존성은 되도록 늘리지 않습니다

## 알아두면 좋은 한계

- 각 서비스의 **비공개 내부 API**에 기대고 있어, 서비스가 바뀌면 깨질 수 있습니다
- 요청은 순차적으로만 보냅니다. 속도를 위해 동시 요청을 넣는 변경은 받지 않습니다(계정이 제한될 수 있습니다)
